# redesign - REDESIGN mode

Audit before touching. Detect sub-mode first.

## Sub-mode

- **Preserve**: modernize without breaking brand. Audit, then targeted evolution.
- **Overhaul**: new visual language, keep content + IA.
- Ambiguous -> ask once: preserve brand or start visually fresh.

## Audit first (before any edit)

Brand tokens, IA, content blocks, existing dial values, SEO baseline (largest redesign risk), a11y wins, analytics events. Record what to preserve vs retire.

## Preserve silently (ask before changing)

Slugs, anchors, nav labels, copy voice, brand logo/wordmark, legal/consent/cookie copy, analytics events.

## Modernize in order

Typography -> spacing/rhythm -> color recalibration -> motion layer -> hero/key-section recomposition -> full block replacement only when unsalvageable.

## Decide depth

Targeted evolution when IA/content/SEO are sound. Full redesign only when visual debt is structural.

## Then run the default loop

Trend pulse (what is current for this kind now) -> Direction (dials from audit, `+1/+2` for overhaul) -> Build -> Critique (gate) -> Verify. Run the gate on the new output, not the old.
